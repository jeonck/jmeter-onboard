# jp@gc - Standard Set 플러그인

jp@gc - Standard Set는 JMeter 플러그인 생태계에서 '표준'이라고 불릴 만큼 필수적인 패키지입니다. 이 세트에는 기본 JMeter만으로는 부족한 정교한 부하 제어와 고급 시각화 기능이 모두 포함되어 있습니다.

실무에서 가장 많이 쓰는 핵심 구성 요소 3가지를 집중적으로 설명해 드릴게요.

## 1. Stepping Thread Group (부하 단계 조절)

기본 Thread Group은 한 번에 모든 사용자를 투입하거나 아주 단순하게만 늘릴 수 있습니다. 반면, 이 플러그인은 **"계단식 부하 주입"**을 시각적으로 설계할 수 있게 해줍니다.

- **기능**: 10명 투입 후 30초 대기 -> 다시 10명 추가 투입 -> 일정 시간 유지 등의 시나리오가 가능합니다.
- **용도**: 서버(t3.medium)가 어느 정도의 사용자 수에서 응답 시간이 급격히 나빠지는지(임계점)를 단계적으로 찾아내기에 최적입니다.

## 2. Transactions per Second (TPS) Listener

부하 테스트의 가장 중요한 지표인 **TPS(초당 처리량)**를 실시간 선 그래프로 보여줍니다.

- **기능**: 서버가 초당 성공시킨 요청 수와 실패한 요청 수를 한 그래프에 그려줍니다.
- **활용**: 사용자를 계속 늘리는데 TPS 그래프가 더 이상 올라가지 않고 평행선을 그린다면, 서버의 자원(CPU, DB Connection 등)이 꽉 찼다는 신호입니다. 이때 Scouter를 보면 CPU 점유율이 100%를 치고 있을 확률이 높습니다.

## 3. Response Times Over Time Listener

시간의 흐름에 따른 응답 시간 변화를 보여줍니다.

- **기능**: 테스트 시작부터 끝까지 응답 시간이 안정적인지, 아니면 점점 늘어나는지 추적합니다.
- **활용**: 테스트 후반부에 응답 시간이 계속 우상향한다면 **Memory Leak(메모리 누수)**을 의심해 볼 수 있습니다. Scouter의 Heap Memory 모니터링 수치와 이 그래프를 비교하면 원인을 정확히 짚어낼 수 있습니다.

## 💡 왜 'Standard Set'인가요?

이 패키지에는 위 3가지 외에도 다음과 같은 유용한 리스너들이 포함되어 있어 종합 선물 세트와 같습니다.

- **Active Threads Over Time**: 실시간으로 몇 명의 가상 사용자가 활동 중인지 확인.
- **Composite Graph**: 여러 그래프(TPS, 응답 시간 등)를 하나로 합쳐서 상관관계를 분석.

## 🛠 설치 및 실습 팁

- **설치**: Plugins Manager에서 jpgc - Standard Set을 체크하고 설치하세요.
- **실습**: t3.medium 인스턴스를 대상으로 Stepping Thread Group을 사용하여 사용자 수를 10명 단위로 계단식으로 늘려보세요.
- **분석**: TPS 그래프가 꺾이는 지점의 시간대를 확인한 뒤, 그 시각 Scouter에 기록된 시스템 자원 상태를 확인하는 것이 부하 테스트 분석의 정석입니다.
