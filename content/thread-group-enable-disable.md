# JMeter Thread Group Enable/Disable 기능

JMeter에서 Thread Group의 Enable(활성화) / Disable(비활성화) 기능은 복잡한 테스트 시나리오를 관리할 때 필수적인 도구입니다. 단순히 "끄고 켜는" 것을 넘어, 실무에서는 전략적으로 활용됩니다.

t3.medium 환경에서 실습하실 때 이 기능이 왜 유용한지 4가지 핵심 이유로 정리해 드립니다.

## 1. 단위 테스트 및 디버깅 (Isolation)

전체 시나리오(로그인→상품선택→장바구니)가 한꺼번에 돌아가면, 어디서 에러가 났는지 찾기 어렵습니다.

- **유용성**: '로그인' 그룹만 Enable하고 나머지(장바구니, 주문)를 Disable 처리하여, 특정 단계가 정상적으로 작동하는지 개별적으로 검증할 수 있습니다.
- **이점**: 문제가 발생했을 때 변수를 최소화하여 원인 파악 속도를 높여줍니다.

## 2. 서버 자원 및 크레딧 관리 (Resource Management)

사용 중인 t3.medium은 CPU 크레딧 기반의 인스턴스입니다.

- **유용성**: 여러 시나리오를 동시에 돌리면 CPU 부하가 급격히 올라가 크레딧이 금방 소진될 수 있습니다.
- **이점**: 현재 집중적으로 테스트해야 할 그룹만 켜두고 나머지는 꺼둠으로써, 제한된 서버 자원을 효율적으로 사용하고 Scouter 모니터링 시 데이터가 섞이는 것을 방지합니다.

## 3. 부하 단계별 조절 (Step-up Testing)

한 번에 100명을 투입하는 대신, 점진적으로 부하를 늘리는 테스트가 필요할 때 사용합니다.

- **유용성**: 
  - Thread Group A (10명) : Enable
  - Thread Group B (50명) : Disable -> 나중에 Enable
  - Thread Group C (100명) : Disable -> 나중에 Enable
- **이점**: 마우스 우클릭만으로 부하의 강도를 실시간으로 조절하며 서버의 한계치(임계점)를 찾을 수 있습니다.

## 4. 다양한 테스트 케이스 보관 (Library)

하나의 .jmx 파일 안에 여러 종류의 테스트(부하 테스트, 스트레스 테스트, 스파이크 테스트 등)를 담아둘 수 있습니다.

- **유용성**: 평상시에는 모든 그룹을 Disable로 해두었다가, 목적에 맞는 그룹만 Enable하여 실행합니다.
- **이점**: 파일 하나로 여러 설정을 관리할 수 있어 형상 관리가 편해집니다.

## 💡 실습 팁: 단축키 활용

- **Disable**: Ctrl + T (또는 마우스 우클릭 -> Disable)
- **Enable**: 다시 Ctrl + T (또는 마우스 우클릭 -> Enable)

비활성화된 요소는 트리 구조에서 **회색(Gray out)**으로 변하며, 테스트 실행 시 아예 무시됩니다.
