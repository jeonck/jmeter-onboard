# JMeter 플러그인 활용

JMeter는 기본 기능만으로도 훌륭하지만, Plugins Manager를 통해 플러그인을 설치하면 분석 능력과 부하 생성 능력이 비약적으로 상승합니다. 마치 순정 자동차에 고성능 튜닝 부품을 다는 것과 같습니다.

가장 대중적이고 실무 활용도가 높은 플러그인 기능들을 정리해 드립니다.

## 1. JMeter Plugins Manager (설치의 시작)

플러그인을 일일이 .jar 파일로 다운로드할 필요 없이, GUI 환경에서 체크박스만으로 플러그인을 관리하게 해줍니다.

**사용법**: Options > Plugins Manager 메뉴에서 필요한 기능을 검색해 설치합니다.

## 2. 핵심 플러그인 종류와 기능

### ① 3 Basic Graphs (시각화의 꽃)

기본 리스너보다 훨씬 정교한 그래프를 제공합니다.

- **Active Threads Over Time**: 테스트 도중 실제 가상 사용자가 몇 명이나 유지되었는지 보여줍니다.
- **Response Times Over Time**: 시간 흐름에 따른 응답 시간 변화를 선 그래프로 그립니다.
- **Transactions per Second (TPS)**: 가장 중요한 지표. 서버가 초당 처리한 트랜잭션 양을 실시간으로 확인합니다.

### ② Custom Thread Groups (정교한 부하 설계)

기본 Thread Group은 부하를 선형적으로만 늘릴 수 있지만, 플러그인을 쓰면 복잡한 시나리오가 가능합니다.

- **Ultimate Thread Group**: 계단식 부하, 스파이크 부하 등 자유자재로 스케줄링이 가능합니다.
- **Stepping Thread Group**: 일정 시간 부하를 주고 유지(Hold)했다가 다시 늘리는 방식의 테스트에 유리합니다.

### ③ PerfMon Metrics Collector (서버 자원 모니터링)

JMeter 안에서 대상 서버(t3.medium)의 CPU, 메모리, 디스크 사용량을 실시간 그래프로 함께 볼 수 있습니다.

**참고**: 서버 측에 별도의 ServerAgent 프로그램을 실행해두어야 통신이 가능합니다. (Scouter와 병행하면 서버 상태를 이중으로 체크하기 좋습니다.)

### ④ Dummy Sampler (스크립트 디버깅)

실제 서버에 요청을 보내지 않고, 내가 설정한 가상의 응답 값(성공/실패/응답 시간 등)을 만들어내는 샘플러입니다.

**활용**: 정규표현식(Correlation)이나 Assertion 로직이 잘 작동하는지 '가짜 데이터'로 미리 테스트해 볼 때 유용합니다.

## 3. 실무 실습 활용 시나리오 (t3.medium 환경)

t3.medium 인스턴스를 테스트할 때 플러그인을 이렇게 활용해 보세요.

- **Ultimate Thread Group**으로 처음 5분은 10명, 다음 5분은 50명으로 부하를 설계합니다.
- 테스트 도중 TPS 그래프를 보면서 서버가 버티는 최대 처리량을 확인합니다.
- 응답 시간이 튀는 지점이 생기면 Response Times Over Time 그래프와 Scouter XLog의 시간대를 대조하여 병목 지점을 찾습니다.

## 💡 플러그인 사용 시 팁

플러그인을 설치한 후에는 JMeter를 반드시 재시작해야 적용됩니다. 또한, 원격 부하 테스트(Master-Slave)를 할 경우 모든 슬레이브 서버에도 동일한 플러그인이 설치되어 있어야 에러가 나지 않습니다.
