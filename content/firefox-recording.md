# Firefox를 활용한 레코딩 설정

Firefox는 JMeter 레코딩 실습에서 가장 권장되는 브라우저입니다. 그 이유는 Chrome이나 Edge 같은 일반적인 브라우저와 달리 '네트워크 설정의 독립성' 때문입니다.

## 1. Firefox가 JMeter 레코딩에 유리한 이유

### 독립적인 프록시 설정:
- Chrome이나 Edge는 윈도우 OS 전체의 프록시 설정을 공유합니다. 따라서 프록시를 켜면 카카오톡, 윈도우 업데이트 등 모든 프로그램이 영향을 받습니다.
- 반면, Firefox는 브라우저 내부에서만 별도로 프록시를 설정할 수 있어, 레코딩 중에 다른 프로그램의 통신이 섞이지 않습니다.

### 인증서 관리의 편리함:
- HTTPS 레코딩을 위해 필요한 JMeter 인증서를 윈도우 시스템 전체에 설치할 필요 없이, Firefox 브라우저 내부에만 간편하게 등록하고 지울 수 있습니다.

### 간편한 스위칭:
- "프록시 사용 안 함"과 "수동 프록시"를 브라우저 설정 창에서 아주 빠르게 전환할 수 있어 실습 효율이 높습니다.

## 2. Firefox 프록시 설정법 (Step-by-Step)

### ① 프록시 주소 설정
1. Firefox 우측 상단의 메뉴(줄 세 개)를 누르고 **설정(Settings)**으로 들어갑니다.
2. 맨 아래로 스크롤하여 네트워크 설정(Network Settings) 섹션의 설정(Settings...) 버튼을 클릭합니다.
3. **수동 프록시 설정(Manual proxy configuration)**을 선택합니다.
4. HTTP 프록시: localhost 또는 127.0.0.1 입력
5. 포트: 8888 (JMeter Recorder와 동일하게)
6. **'모든 프로토콜에 이 프록시 서버 사용'**에 체크합니다.
7. 확인을 눌러 저장합니다.

### ② HTTPS 인증서 등록 (보안 사이트 레코딩 필수)
이 작업을 안 하면 https://로 시작하는 사이트 접속 시 "연결이 안전하지 않음" 에러가 납니다.

1. JMeter의 HTTP(S) Test Script Recorder에서 [Start] 버튼을 먼저 누릅니다. (이때 bin 폴더에 인증서 파일이 생깁니다.)
2. Firefox 설정에서 개인 정보 및 보안(Privacy & Security) 탭을 클릭합니다.
3. 맨 아래 인증서(Certificates) 섹션에서 **인증서 보기(View Certificates...)**를 누릅니다.
4. **가져오기(Import...)**를 누르고 JMeter 설치 폴더의 bin/ApacheJMeterTemporaryRootCA.crt 파일을 선택합니다.
5. **"이 인증 기관을 믿고 웹사이트 신원을 확인(Trust this CA to identify websites)"**에 체크하고 확인을 누릅니다.

## 3. 레코딩 실습 팁

- **레코딩이 끝나면**: 반드시 Firefox 설정을 다시 **'프록시 사용 안 함'**이나 **'시스템 프록시 사용'**으로 돌려놓아야 일반적인 인터넷 서핑이 가능합니다.
- **캐시 삭제**: 레코딩 직전에 Firefox의 방문 기록과 캐시를 비워주세요. 그래야 브라우저가 서버에 새로 요청을 보내는 과정이 JMeter에 정확히 기록됩니다.
