# Block Coding Approach

JMeter를 '블록코딩(Block Coding)' 또는 '레고(Lego) 조립' 방식이라고 부르는 이유는, 소스 코드를 직접 한 줄씩 타이핑하는 대신 기능 단위의 컴포넌트(요소)들을 트리 구조로 배치하고 설정하는 방식으로 테스트 시나리오를 만들기 때문입니다.

구체적인 이유는 다음과 같습니다.

## 1. Visual Hierarchy

JMeter의 왼쪽 패널을 보면 모든 로직이 계층적인 트리 형태로 구성됩니다.

마치 스크래치(Scratch)나 엔트리 같은 블록코딩 도구처럼, '상위 요소(부모)' 아래에 **'하위 요소(자식)'**를 끼워 넣는 식입니다.

예를 들어, "로그인 성공 시(상위) -> 마이페이지 이동(하위)"과 같은 로직을 코드가 아닌 마우스 우클릭과 드래그로 구현합니다.

## 2. Drag & Drop + UI Configuration

복잡한 프로그래밍 문법을 외울 필요가 없습니다.

- **로직 컨트롤러(Logic Controller)**: If Controller(만약 ~라면), Loop Controller(반복) 등의 블록을 가져다 놓습니다.
- **설정**: 블록을 클릭하면 나타나는 UI 창에서 숫자나 조건식만 입력하면 됩니다.

소스 코드 기반의 도구(예: Locust, Gatling)는 오타 하나에도 작동하지 않지만, JMeter는 준비된 블록을 조합하기 때문에 문법 오류에서 상대적으로 자유롭습니다.

## 3. Modular Components

JMeter의 각 요소들은 독립된 기능을 가진 **'레고 블록'**과 같습니다.

- **Sampler**: 요청을 보내는 블록 (HTTP, FTP, JDBC 등)
- **Config Element**: 변수나 헤더를 설정하는 블록
- **Assertion**: 성공/실패를 판정하는 블록 (결과 검증)
- **Listener**: 데이터를 수집해 그래프로 보여주는 블록

이 블록들을 어디에 배치하느냐에 따라 적용 범위(Scope)가 결정되는 직관적인 구조를 가집니다.

## 4. Is Code Writing Completely Impossible?

아닙니다. '블록'만으로 해결되지 않는 복잡한 로직(예: 특수한 암호화 처리)이 필요할 때는 BeanShell이나 JSR223 Sampler라는 특별한 블록을 추가하여 그 안에 직접 자바(Java)나 그루비(Groovy) 코드를 써넣을 수도 있습니다.

한마디로 정리하자면: "복잡한 성능 테스트 로직을 코딩 없이 컴포넌트 조립만으로 설계할 수 있기 때문에" 블록코딩 방식이라고 부릅니다.
